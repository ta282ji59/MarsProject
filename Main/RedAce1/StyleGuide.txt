## コーディング規約

・共通
    - 極力各コーディング規約や命名規則の従う
    - 一文字の変数や一般的でない略語は避けるべき
    - 名前が説明的である（名前から意図が読み取れる）こと
    - 名前から予想できる以上の処理を極力行わない
    - 配列は複数形か"list"などつける

・HTML
    - インデントはスペース2つ
    - 属性値にはダブルクオテーション
    - クラス名はケバブケース(チェインケース) kebab-case
    - ファイル名はスネークケース　snake_case

・CSS
    - インデントはスペース
    - h div p等のタグをセレクタに指定しないで、必要に応じてクラスを割り振る
    - 基本、要素セレクタに対して装飾しない
    - クラス名はケバブケース(チェインケース) kebab-case
    - ファイル名はスネークケース　snake_case
    - クラス名の単語候補、https://qiita.com/manabuyasuda/items/dbb76ed36970bec95470

・JavaScript
    - インデントはスペース4つ
    - 1行は80文字
    - 常に演算子（ = + - * / ）の前後と、カンマの後にスペース
    - 開き括弧の前にスペースを1つ
    - 新しい行に、スペースを入れずに閉じ括弧
    - 比較演算子は==ではなく===を使用
    - 変数と関数名は ローワーキャメルケース (camelCase)
    - クラス名は アッパーキャメルケース（CamelCase）
    - グローバル変数は大文字
    - 定数は大文字のスネークケース (SNAKE_CASE)
    - 変数は名詞の形、メソッド名は動詞の形
    - ファイル名は小文字のみ、区切り文字はアンダースコア (_) 

・Python
    - ファイル名は小文字のみ、区切り文字はアンダースコア
    - インデントはスペース4つ
    - 演算子の前後はスペース1つ
    - 括弧やブラケットの前後に空白を入れない
    - 行の長さは最大79文字
    - import文は行を分ける（from ... import は同じ行に書いてOK）
    - モジュールの名前は小文字
    - クラス名は CapWords 方式（AbcDefg）
    - 型変数の名前は、最初が大文字で短く（Num）
    - 関数の名前は小文字のみ
    - インスタンスメソッドのはじめの引数の名前は self
    - クラスメソッドのはじめの引数の名前は cls
    - 変数は小文字、定数は大文字
    - Python の文法チェックツールである flake 8 を活用
